## 一、产品概述

### 1.1 产品定位

学术会议通用管理平台是一套面向学术会议承办方的 **SaaS 化全流程管理解决方案**，覆盖会议创建、报名缴费、住宿预订、论文投稿评审、日程编排、现场签到、展商管理等完整业务链路，同时提供面向参会人员的宣传展示端（Web + H5 微端），实现"管理 → 宣传 → 报名 → 参会 → 签到"的业务闭环。

### 1.2 目标用户

| 角色 | 说明 |
|------|------|
| 平台管理员 | 管理系统级配置、用户、角色和权限 |
| 会议承办方（商家） | 创建并管理学术会议的全流程 |
| 审稿专家 | 对投稿论文进行评审打分 |
| 参会人员（C端用户） | 注册登录、报名缴费、投稿、预订住宿、查看日程等 |
| 展商 | 申请参展、提交展位信息 |

## 二、核心业务流程

### 2.1 会议全生命周期流程

```
创建会议 → 配置会议（费用/酒店/模板/类型）→ 发布会议
    ↓
宣传展示端上线 → C端用户注册 → 报名缴费
    ↓
论文投稿 → 初审 → 稿件评审 → 录用/退稿
    ↓
日程编排（会场 + 时间段）→ 发布日程
    ↓
住宿酒店预订 → 缴费确认
    ↓
展商申请 → 审核入驻
    ↓
现场签到 → 数据统计
    ↓
会议归档
```

### 2.2 参会报名流程

```
用户注册/登录 → 选择会议 → 填写报名信息 → 选择缴费项目
    ↓
在线支付报名费 → 生成报名订单 → 报名成功
    ↓
（可选）选择住宿酒店 → 选择房型 → 缴费 → 生成住宿订单
    ↓
（可选）提交论文投稿 → 缴费 → 等待审核
    ↓
获取参会凭证 → 现场签到
```

### 2.3 论文投稿评审流程

```
投稿人提交论文 → 缴纳投稿费
    ↓
管理端收到投稿 → 初审（形式审查）
    ↓
初审通过 → 进入稿件评审池
    ↓
分配审稿专家 → 专家评审打分
    ↓
评审结果：录用 / 修改后再审 / 退稿
    ↓
通知投稿人 → 录用论文纳入会议日程/论文集
```

### 2.4 住宿预订流程

```
管理端配置酒店及房型 → 关联到具体会议 → 设置库存及价格
    ↓
C端用户在报名时/报名后 → 选择酒店 → 选择房型
    ↓
提交预订 → 在线缴费 → 生成住宿订单
    ↓
管理端查看预订数据 → 与酒店对接安排
```

### 2.5 展商入驻流程

```
管理端配置展商申报信息（条件/费用/展位等）
    ↓
展商通过 C端入口提交申请 → 填写企业信息 → 提交材料
    ↓
管理端审核 → 通过/驳回
    ↓
通过 → 展商信息展示在宣传端
```

## 三、系统一：学术会议管理系统（B端）

### 3.1 主页（Dashboard）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 数据统计总览 | 人员统计（报名人数、参会人数、VIP人数）、学术统计（投稿数、日程数、分会场数）、住宿酒店统计（酒店数、预订数、房间数） | P0 |
| 快捷入口 | 可自定义的常用功能入口，如"创建会议""查看报名""审核投稿"等 | P1 |
| 待办提醒 | 待审核投稿、待处理报名、待审核展商等通知 | P1 |

### 3.2 会议管理

#### 3.2.1 会议配置

| 配置项 | 说明 |
|--------|------|
| 住宿酒店配置 | 选择并关联已维护的酒店及房型信息 |
| 报名费用配置 | 设定不同参会类别的费用（如早鸟价、正常价、学生价） |
| 宣传展示端模板配置 | 选择/自定义 C端宣传页面模板 |
| 会议类型配置 | 行业分类/会议规模/会议级别等 |

#### 3.2.2 会议发布

| 功能 | 说明 |
|------|------|
| 基础信息 | 会议名称、时间、地点、主办/承办/协办单位、简介等 |
| 参会报名费用 | 关联费用配置，支持多级费用 |
| 住宿酒店选择 | 从已配置酒店中选择并设置库存 |
| 论文投稿要求 | 投稿主题、格式要求、截止时间、投稿费用 |
| 默认创建主会场 | 会议发布时自动创建主会场 |
| 发布状态控制 | 草稿 → 待审核 → 已发布 → 已结束 → 已归档 |

#### 3.2.3 会议列表与操作

| 功能 | 说明 |
|------|------|
| 会议列表 | 分页展示，支持按名称/时间/状态/类型搜索 |
| 审核 | 对待审核会议进行审批（通过/驳回） |
| 编辑 | 修改会议信息（未发布/已发布状态均可） |
| 查看 | 查看会议详情及关联数据 |
| 统计 | 查看单个会议的报名/投稿/住宿等维度统计 |

## 五、数据实体关系（核心）

```
会议(Conference)
  ├── 会场(Venue)          1:N
  │     └── 日程(Schedule)   1:N
  ├── 报名(Registration)    1:N
  │     └── 订单(Order)       1:1
  ├── 投稿(Submission)      1:N
  │     └── 评审(Review)      1:N
  ├── 住宿安排(Accommodation) 1:N
  │     ├── 酒店(Hotel)        N:1
  │     ├── 房型(RoomType)     N:1
  │     └── 预订(Booking)      1:N
  ├── 展商(Exhibitor)       1:N
  ├── VIP(VIPGuest)         1:N
  └── 签到(CheckIn)         1:N

用户(User)
  ├── 角色(Role)            N:N
  └── 权限(Permission)      N:N（通过角色间接关联）
```

## 六、非功能需求

| 维度 | 要求 |
|------|------|
| 多端适配 | 管理端 Web 响应式；C端 Web + H5 微端适配 |
| 支付集成 | 微信支付 + 支付宝（报名/投稿/住宿） |
| 通知系统 | 短信 + 邮件通知（报名确认/审核结果/日程提醒等） |
| 文件上传 | 论文 PDF/Word 上传，酒店/景点图片上传 |
| 数据安全 | 用户敏感信息加密存储，HTTPS 传输 |
| 角色权限 | RBAC 权限体系，支持细粒度功能+数据级别权限控制 |
| 性能 | 首页加载 < 3s，列表接口响应 < 500ms |
| 可扩展性 | SaaS 架构，支持多会议、多商家并存 |
